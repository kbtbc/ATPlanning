# Resupply UI Redesign - Design Mockups

## Current Problems Identified

1. **Confusing Information Hierarchy** - When tapping a resupply point, users see a jumble of services, contacts, and notes without clear visual hierarchy
2. **Unclear Service Categories** - Icons for Grocery, Post Office, etc. don't clearly communicate what's available vs. unavailable
3. **Random Contact Listings** - 3 businesses shown seem incomplete and disconnected from context
4. **Outdated Summary Stats** - The 51/45/13 (Full/Limited/Minimal) doesn't help users make decisions
5. **Weak Navigation** - Relationship between Upcoming list and Contact Directory is disjointed
6. **Visual Noise** - Too many colored badges, icons, and text competing for attention

---

## Design Approach: "Trail-Focused Simplicity"

**Core Principle**: A thru-hiker needs to answer ONE question: *"Where can I get what I need, and how do I get there?"*

---

## Recommendation: HYBRID APPROACH

1. **From Mockup C**: Consolidated smart list as primary view
2. **From Mockup B**: Service-first filtering as optional mode
3. **From Mockup A**: Timeline header for journey context
4. **From All**: Progressive disclosure - simple by default, detailed on tap
